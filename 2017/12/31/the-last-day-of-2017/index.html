这次的年终总结，我是认真的

不得不说，2017年，就这么过去了。

记得去年的这个时间我也写了篇年终总结，地址在这里，现在翻过来重新打开看，觉着当时自己蛮搞笑的，只怪自己太年轻，没能意识到现实的残酷以及社会竞争的激烈，所以我想借着今天这篇总结，也把去年没说的也一并说了。

就在去年，我刚毕业，装着在学校学的半吊子编程水平踏入深圳这片土地，印象记得自己是在学校的时候企业校招过来的，当时共有130多号人去面试+笔试这家公司，因为一些不可明说的原因最终要了8个人而自己则刚好是运气加上吹嘘的实力也在其中之一，负责招聘负责人说我去公司做 Android 开发[外加另外两个共三个人]，因为本身在学校学了一些这块皮毛，而当时手头又没有其它更满意的 offer，所以就签了三方，于是16年3月份就按计划去了公司报到。

去之前我是满脑子的憧憬呀，想着我技术比较菜，去公司了肯定有大牛，嗯，没错，只要勤于编码，多向大牛请教，假以时日，必成大拿，接着迎娶白富美，坐享人生巅峰，这梦做的… 可是啊，去了公司后才发现根本就没有 APP 开发项目组，公司主营都是 Web 项目，招聘时说做 Android，只是因为领导想把Web 端的一些业务往移动端发展，让我们几个先做个 Demo 出来…我这吐的一口老血：一没有产品，二没有需求，三没有技术带头人，我们几个就照着给的一个旧时的文档上的来。

美工，切图，NO，不存在的，没有图，就自己抠自己画，色彩自己取色，技术不懂，自己百度+goolge，捯饬捯饬几个月后，产品是出来了，按照 boss 说的，功能啥的都有，交给上头领导，嗯，没有下文，boss说再接着做另一个 Web 端项目的 Demo 出来，好吧，继续捯饬，最终也出来了，交给上面，是的，你 没猜错，依然没下文。

也就是说很长一段时间，我都是吃的老本，我的编程水平并没有进步，而那两个完成的 Demo，由于工程开发本身就没有什么项目规范，也没有什么技术框架，仅仅只是符合了 boss 的功能要求，所以我估计后来人维护起来挺困难的[我现在已经不在那家公司了]，毕竟只是Demo级别的东东。

你问我为什么不走！走哪儿去，就这水平，一没工作经验，二没技术能力，出去谁要啊，在公司起码还能养活自己，学点东西，那会儿自学了Android5.0新特性，跟进AS工具的持续迭代版本更新，还接着关注了郭神、弘神、医生、张哥的博客，买了一些书。看到这里，你以为我技术有了很大进步了，并没有，原因嘛我也不知道，现在仔细分析，我觉得就是学的太杂了，那段时间我的确接触的很多新知识点，但并不能熟练运用，公司让指哪就打哪，所以在公司我们几个一直是处于「游离组」的状态，对于公司而言，是没有产出的，以至于去年都么有所谓的年终奖。

对了，我还一度被公司外派到外面的项目组协助，在那里，我们是乙方，在那里，我体会了内部员工和外部员工的不同，体会了甲方和乙方的待遇差别。人家吃着内部食堂，我们要走路去外面吃，外面并没有多少吃饭的餐馆，我差不多吃了一个月的兰州拉面..;人家到点准时下班，我们不可以，我知道有个别同事天天加班到晚上11、12点，改着甲方口头上随时变更的需求…..。然而这都不是事，那边APP组依然没有一个完整的项目组，就我一个，加上甲方的一个leader，我依旧单枪匹马，学着所谓的Hybrid APP开发技术，是的，我认真学了，我认为我把问题解决了，遗憾的是并没有在他们规定的时间内解决，甲方认为我经验还不够..于是乎，我又回到了公司本部，继续做着一些不痛不痒的项目..

终于，时间往前推了一年，到了17年5月份，在这公司待满了一年零三个月十八天，我觉着是时候走人了，再留在公司也没意思，下决定裸辞了。

很多人说16年是找工作寒冬季，是的，整个大环境都不是很好，特别Andrid市场，趋于饱和，用人单位要求提高了，基于当时的情况，我果断转Java 开发，陆续看了一点视频，边看边投简历，我清楚的记得，我17年5月10号离职的，11号就有公司要我面试，于是就跑去笔试+面试，13号有公司说让我入职，就是我现在的公司，你可能会问为什么不再等等，兴许有更好的公司，出于对自己的不自信，万一迟迟没有找到新工作，我的基本生活就会成问题，我认为有公司要我就不错了。你可能又要问，我靠你怎么那么快就有面试机会，你错了，虽然我是5月份离职的，但更早的2、3月份我就在准备了，改简历，投简历，几个月下来，也就收到3、4份面试机会而已。

说到现在这家公司，你要说他有多好并没有，但对我来说能进一个结构完整的项目组就非常不错了，我也是进来公司才知道，由于是新项目，这家公司在大批量招初级工程师和少量的中级高级工程师，所以刚好我又是幸运吧，来公司的这半年，技术不能说有多大提升，起码让我见证了一个项目的起始发展到功能演进到最终项目上线，整个项目的结构有项目经理、产品、运维、前端、后端、UI、测试团队(本地环境，测试环境，UAT环境)、技术总管以及客户方，而项目本身的规范也是以前没有机会实践的，从发布、部署、代码规范、提交、需求更改、bug录入等各个方面可以说是让我涨见识了，看到这里你肯定会嘲笑[捂脸哭]。是的，我之前没有接触过这些。

谈到项目，就不得不提与之相关的技术了，项目看着高大上(为避免打广告之嫌，就不说名了)，基于外国人开发的架构，用的却是很老的技术，多老呢，XPATH、xsl、struts1、xml…哈哈，我估计除了struts1和xml，其它的你是不是没听过，不过这并不重要，毕竟技术年年换，学习能力才是香饽饽，能跟同组内比我优秀的大佬一起吹逼才好玩，问他们问题，看他们写的代码，观察他们改代码的严谨风格…都让我收获颇丰。

在这一年里，我关注的不光有技术，我开始了解接触投资理财，付费加入一些圈子，关注不同类型的优质公众号，搭建个人博客，断断续续写了些文章和笔记，因为我始终认为技术不是唯一，对于很多人来说，思维的局限，认知的不足是成长路上最大的瓶颈。

通过系统学习理财，让我避免很多理财陷阱和骗局，从而更好的约束自己，控制收支，理财观念的深入，让我意识到钱是可以生钱的，富人玩的是资金流，而穷人只知道存银行，在通货日益膨胀的今天，理财虽不能发家致富，却能让资产保值[/(ㄒoㄒ)/~~虽然钱包没有多少钱]。

内容付费时代，通过加入一些有价值的圈子，让我有机会旁观同行各业形形色色人物状态，看他们分享各自学习工作生活所思所获，或赞同或反对，对我有用的就吸收，没用的就忽略。

关注内容公众号写手，通过持续输入优质原创文章，改变了我的阅读习惯，提升了我的阅读质量，拓宽了思维空间，大大增加我获取有效信息的来源，同时也能保证写文章有效输出。

啊，不知不觉已经写了这么多了，感觉这篇不应该叫年终总结，而应该改叫年终回顾。回忆完了过去的精彩[糟糕]片段，按台本应该规划来年的工作状态了。

每个人都会制定计划，但不是每个人都能有效执行计划，在2017年结束倒计之时，正式迈入2018年之即，我知道朋友圈又将掀起新一轮轰轰烈烈的誓师大会。”新的一年，不一样的自己，力争今年完成XXX人生大事”，”今年我要减肥，瘦成一道闪电”，”今天我要实现收入翻倍”。每个人都忙着制定新年计划，热血沸腾的加入辞旧迎新的行列。

一周后，多数人将会精彩开挂：健身卡办起来，心仪已久的在线课程报起来，饮食起居规律起来…差点就被自己感动了。

遗憾的是，两个月后，三个月后，多数人发现原来2018年还是那个矬样，自己还是跟以往的2017年没有太大的差别，没有变成全新的自己，计划也永远只是停留在计划上，就像我一样，说好了17年每天花一点时间好好学英语，坚持下来，结果，和如上所说相反，我真的坚持下来了，像我这种不按套路出牌的怎么可能会失败呢！我重新拾起英语，从头开始学，我坚信，这将会为我以后职场打开一条更宽阔的道路，所以，坚持吧少年。

技术方面，目前，各种新新概念层出不穷，什么大数据、人工智能、AR、区块链，各种鼓吹预测下一个风口，让你眼花缭乱，”耳不暇接”，对此，我不盲目跟从，也不抵触，也没有精力去整那么多事儿，眼下提升自己工作技术能力是为第一要务，适当往全栈方面拓展，以后的竞争无疑更加激烈，用人单位对员工要求也是越来越高，接下来的一年也是职场生涯中比较关键的一年，所以，加油吧少年！

最后，祝愿每位读者在新的一年里，都能借助自己牛逼的计划，成就牛逼的自己。
