更换了电脑如何更新博客

一晃差不多一两个月没有更新博客了，一方面，这东西的确需要坚持写作，另外笔者遇到的情况是 之前用于发布博客的那台电脑被我卖了(太卡了，唉！说多了都是泪….)，而我又辞了工作，这样一来手里头就没本本了，也就没怎么更新了。后来找着了新工作，陆续入职。也一直提醒着自己博客不能停，毕竟搭建起来花费了不少精力，一定要坚持，就这样拖到现在，随便一折腾时间就过去一大半了

好了话说回来，换了电脑如何继续更新博客呢，要确保之前的配置文章之类的不变，网上搜了好久，发现如下方法执行起来还是可以的，方便、可靠。

  1. 在新电脑重新安装 git，可以去官网下载

  2. 重新安装 Node.js

    windows 32 位安装包下载地址

    windows 64 位安装包下载地址

  3. 安装 hexo，在电脑桌面–>右键–>git bash here，键入如下命令

    npm install -g hexo

  4. 接下来就将原来的文件拷贝到新电脑中(在之前的电脑卖出前我已将博客文件夹下所有文件全部备份到了优盘，如下图)，分如下步骤：

  • 类似之前一样，我在这台新电脑 E 盘也建了一个 myblog 文件夹，把之前备份的博客文件夹下的_config.yml，themes，source，scaffolds，package.json，.gitignore 拷贝到新电脑的 myblog 文件夹下
  • 在 E 盘 myblog 文件夹下右键–>git bash here，键入npm install命令进行模块安装

注意 这里我们没有用hexo init初始化初始化，因为有的文件我们已经拷贝生成过来了，所以不必用hexo init去整体初始化，如果不慎在此时用了hexo init，则站点的配置文件_config.yml里面内容会被清空使用默认值，切记切记

  • 进入 myblog 根目录下，敲如下命令，为了使用hexo d命令来部署到git上

    npm install hexo-deployer-git --save
  • 好了，测试一下，现在新建一篇文章，编辑完了然后敲hexo g -d看是不是部署成功了！
